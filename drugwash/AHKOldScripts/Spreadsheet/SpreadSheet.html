Add control to the Gui

Parameters

hGuiParent’s hwnd
x-hControl coordinates
styleWhite separated list of control styles, by default VSCROLL HSCROLL
handlerNotification handler, optional

Styles

VSCROLL HSCROLL STATUS GRIDLINES ROWSELECT CELLEDIT GRIDMODE COLSIZE ROWSIZE WINSIZE MULTISELECT

Handler

result := Handler(hwnd, event, a1, a2, a3)
hwndHandle of the speradsheet control that sends notification
eventEvent that ocured.  Can be “select”, “edit”, “update”, “button”, “hyperlink”
etypeEvent type.  Depends on event.  Can be “before”, “after”, “button”, “hyperlink” or number.  See bellow.
colColumn of the sected cell.
rowRow of the selected cell.
resultHandler result, depends on event.  See bellow

Event types and returns

etype parameter of the handler is specific to each event:

selectcurrent splitt window number.
edit”before” or “after”.  Return 1 to prevent entering editing (before) or to discard changes (after).
update”before” or “after”.
click”button” or “hyperlink”.

Returns

Control handle

GetCellText

SS_GetCellText(hCtrl,
col,
row)

Get cell text

Remarks

This function retrieves any kind of text from all types of cells, no matter the internal representation (i.e. integer, string, double precision floats will be converted to AHK string) GetCell function, contrary, behaves differently.  For instance, it will return currently selected item in ComboBox cell type, unlike this function which returns text of the selected item.  The function can’t return graph definition, the same as GetCell.

Type dependent parameters

txtNumber (INTEGER), hwndCombo (COMBOBOX), formula definition (FORMULA), graph definition (GRAPH)

Types

  • TEXT, TEXTMULTILINE, INTEGER, FLOAT, FORMULA, GRAPH, HYPERLINK, CHECKBOX, COMBOBOX, EXPANDED, BUTTON, WIDEBUTTON, EMPTY, COLHDR, ROWHDR, WINHDR
  • FIXEDSIZE - Will force a 15 by 15 pixel image.  To be combined with BUTTON, CHECKBOX or COMBOBOX
  • FORCETYPE - The cell will preserve its type when edited.  To be combined with TEXT, TEXTMULTILINE, BUTTON, WIDEBUTTON or HYPERLINK

Type combinations

BUTTONcan be combined with TEXT or TEXTMULTILINE
WIDEBUTTONcan be combined with TEXT or TEXTMULTILINE
FORCETYPEcan be combined with TEXT, INTEGER, FLOAT, TEXTMULTILINE, BUTTON, WIDEBUTTON or HYPERLINK
FIXEDSIZEcan be combined with BUTTON, CHECKBOX or COMBOBOX

States

LOCKED HIDDEN REDRAW ERROR DIV0 UNDERFLOW OVERFLOW RECALC ERRMASK

Aligment

Graphs

Syntax

Grp( [T], X, Y, fx1 | gx1, fx2 | gx2, ...)      ;no spaces allowed

Definition

T(xp,yp,Orientation,Color,”Text” )Text (max 16)
X,Y(xMin,xMax,xOrigo,xStep,Color,”Text”)X and Y axis (max 1)
fx(function,xStep,Color,”Text”)Graph from math function (max 16)
gx(r1:r2,Color,”Text”)Graph from cell values (max 16)

Note

Bad graph definitions will burn CPU or crash application.  Even single space more in Grp syntax will do that, and you will have to kill the AutoHotKey.exe using task manager.

Examples

First Sample

#Singleinstance, force
    Gui, +LastFound +ToolWindow
    hwnd := WinExist()

    Gui, Add, ListBox, x0 y0 w0 HWNDhList, MyButton||
    hCtrl := SS_Add(hwnd, 0, 0, 300, 400, "VSCROLL  GRIDMODE CELLEDIT ROWSIZE COLSIZE ROWSELECT")
    SS_SetRowHeight(hCtrl)

    SS_SetColCount(hCtrl, 2)
    SS_SetRowCount(hCtrl, 8)

    SS_SetColWidth(hCtrl, 1, 148)
    SS_SetColWidth(hCtrl, 2, 145)

    SS_SetGlobalFields(hCtrl, "cell_txtal", "CENTER MIDDLE")

    hCombo := SS_CreateCombo(hCtrl, "item1|item2|item3|item4")

    SS_SetCell(hCtrl, 1, 1, "type=TEXT", "txt=Caption", "bg=0xFF", "fg=0xFFFFFF")
    SS_SetCell(hCtrl, 2, 1, "type=TEXT", "txt= ")

    SS_SetCell(hCtrl, 1, 2, "type=TEXT", "txt=Style", "bg=0xFF", "fg=0xFFFFFF")
    SS_SetCell(hCtrl, 2, 2, "type=COMBOBOX", "txt=" hCombo, "data=1", "imgal=RIGHT",  "txtal=CENTER")   ;select 2nd item

    SS_SetCell(hCtrl, 1, 3, "type=TEXT", "txt=Anchor", "bg=0xFF", "fg=0xFFFFFF")
    SS_SetCell(hCtrl, 2, 3, "type=BUTTON TEXT", "txt=w0.5 h", "imgal=RIGHT", "txtal=CENTER")

    SS_SetCell(hCtrl, 1, 4, "type=TEXT", "txt=Visible", "bg=0xFF", "fg=0xFFFFFF")
    SS_SetCell(hCtrl, 2, 4, "type=CHECKBOX", "data=1", "imgal=CENTER")

    SS_SetCell(hCtrl, 1, 6, "type=TEXT", "txt=Help", "bg=0xFFFF", "fg=-1")
    SS_SetCell(hCtrl, 2, 6, "type=HYPERLINK", "txt=www.autohotkey.com", "txtal=CENTER")

    SS_SetCell(hCtrl, 1, 8, "type=WIDEBUTTON TEXT", "Txt=Wide &Button", "txtal=CENTER")

    Gui, Show, w300 h150
return

Graph Example

    Gui, +LastFound
    hwnd := WinExist()

    hCtrl := SS_Add(hwnd, 0, 25, 552, 477, "WINSIZE VSCROLL HSCROLL CELLEDIT ROWSIZE COLSIZE STATUS MULTISELECT")

    graph =
    (LTrim Join
        Grp(
            T(-1,0,0,Rgb(0,0,0),"Graph Demo"),
            X(0,PI()*4,0,1,Rgb(0,0,255),"x-axis"),
            Y(-1.1,1.1,0,0.5,Rgb(255,0,0),"y-axis"),
            gx(AJ1:AJ13,Rgb(0,0,0),"Cell values"),
            fx(Sin(x()),0.1,Rgb(255,0,255),"Sin(x)"),
            fx(x()^3-x()^2-x(),0.1,Rgb(0,128,0),"x^3-x^2-x"))
    )

    SS_SetCell(hCtrl, 1, 1, "type=GRAPH", "txt=" graph, "bg=0x0D0FFFF")
    SS_ExpandCell(hCtrl, 1, 1, 6, 15)

    Gui, Show, w550 h500, SpreadSheet
return

Known Bugs

  • Cell cursor disapears in expanded cells.
  • Multiselect scrolling by mouse is too fast.
  • Scroll-locked area does not work well with splitts.
  • Float to ascii does not work on numbers > 1e+4000 or < 1e-4000
  • Very slow if rows>10000
  • If you doubleclick the cell, even if you forbid editing values or selection, it will still enter it for text fields.
  • 0.0.1.5 : ComboBox doesn’t work from AHK for some reason (KetiO reported that he changed creation flags for listview).
  • 0.0.1.5 : Row header can be edited and after that it loses rowhdr type.
